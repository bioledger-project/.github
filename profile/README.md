# BioLedger

Building open tools for provenance-aware scientific computing, reusable research assets, and reproducible analysis workflows.

Find examples and documentation here: https://bioledger-project.github.io/bioledger

## Vision

Scientific computing should be portable, reproducible, and independent of any particular platform.

BioLedger is an open ecosystem for describing scientific data, tools, and analyses as durable, machine-readable artifacts that can be shared, validated, reproduced, and reused across environments.

Rather than treating workflows as the starting point, BioLedger is designed around how researchers actually work: through exploration, experimentation, and iteration.

## Why BioLedger?

Modern scientific software often ties users to specific workflow engines, platforms, or execution environments.

BioLedger takes an artifact-first approach.

### Structured data

Metadata standards such as ISA-Tab help transform datasets from collections of files into reusable research assets that both humans and AI systems can understand.

### Portable tools

Language- and execution-agnostic tool specifications allow analyses to run locally or integrate with existing workflow systems without locking users into a particular platform.

### Reproducible analyses

Researchers should be free to experiment. BioLedger records what happened and generates provenance artifacts, workflows, and RO-Crates from successful analyses rather than requiring workflows to be designed up front.

### AI where it helps

Large language models are used to assist with tasks such as metadata generation, planning, and tool discovery. The source of truth remains structured, versionable artifacts that can be independently verified and reproduced.
